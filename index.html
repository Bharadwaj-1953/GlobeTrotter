User Research

Understanding how college students really plan trips

We conducted in-depth interviews with 6 college students on February 4, 2026. Different travel styles. Different budgets. Different confidence levels. All facing the same core problems.

🔄

Platform Fragmentation

Students repeatedly switch between Google Flights, Expedia, Airbnb, and more. Each platform shows different information, making comparison impossible without tedious manual work.

⏱️

Time Pressure

Between classes, work, and other commitments, students have limited time. Many accept higher costs or shorter trips just to save planning time.

👨‍👩‍👧‍👦

Group Coordination Nightmare

Group travel decisions drag on. Different budgets, conflicting schedules, delayed responses. Group chats become chaotic. Someone always feels like they compromised.

Design Philosophy

Simplicity meets functionality

Clarity First

Every button, every piece of information, every action has a clear purpose. Students don't have time for confusion.

Visual Hierarchy

Important information stands out. Blue conveys trust and reliability. Teal highlights actions. Clean typography aids scanning.

Reduced Cognitive Load

Generous whitespace. Smart grouping. Progressive disclosure. Features appear when needed, not all at once.

Consistent Patterns

Cards, buttons, filters work the same everywhere. Users learn once, apply everywhere. Predictable is good.

Feedback Matters

Changes are confirmed. State is communicated clearly. Users always know what happened or what will happen next.

Mobile Ready

Responsive design from the ground up. Touch-friendly interactions. Works seamlessly on phones, tablets, desktops.

Limitations

  • Uses static mock data, not connected to real APIs
  • Booking and payment processing not implemented
  • Group interactions are simulated, not dynamically synchronized
  • No real-time backend updates or authentication
  • This is a design prototype, not a production app
